Open Guide to AWS

在 Twitter 上看到這個：The Open Guide to Amazon Web Services

心裡想的就是：『把我想做的都做玩了』。我在整理 AWS Study Roadmap by Categories 的時候，也想整理過類似的東西。

把他的目錄條列出來，前面有一些資訊很重要，特別說明。

基本資訊

  • Common Concepts：基本的參考資訊、API Endpoints、ARNs (AWS Resounce Naming) 等基本概念。這其實還滿重要的，就像一般學程式語言的第一章，通常很重要，但是都很容易被忽略。
  • Servie Matrix：跟其他 IaaS 比較：用通用名稱，像是 PasS 取代 Beanstalk 這種專有名詞，同時和其他 IaaS 比較，像是 GCP 和 Azure。
  • AWS 產品線的生死 (Release and Maturity)：有些產品線已經快要退出江湖了，像是 SimpleDB
  • Compliance (遵循)：通常是一些安全性規範，像是 PCI DSS、SOC …
  • Getting Help and Support：如何取得協助的管道，包含 Forum、Support、Account Manager … 等
  • Restrictions：使用的限制
  • Certifications：教育認證制度
  • Managing AWS
    • Managing Infrastructure State and Change
    • AWS Configuration Management
    • AWS Console
    • Command-Line tools
    • APIs and SDKs
    • Boto
    • General Visibility
  • Managing Servers and Applications
    • AWS vs Server Configuration
    • Philosophy
    • Server Configuration Management
    • Containers and AWS
    • Visibility

Boto3 and Operaions

裡面提到，使用 boto3 取代 bash，因為比較好處理回傳的結果。

這點跟我正在公司裡推的想法是一樣的。同時除了 boto3，我也覺得針對 Operation 相關的任務，Python 會比 CLI or Node.js 都適合。 non-blocking 不適合做程序性的任務，Bash 不擅長做字串處理，python 同時可以滿足這兩個，同時又可以 OOP，是 Operation 的首選。
