Whitepaper - AWS Well Architected Framework

整理 AWS Well Architected Framework, October 2015 的重點，前言提到以下重點：

  • 了解架構設計的利弊 (pros and cons)
  • AWS 根據多年的經驗，同時 review 數千個各式各樣的客戶案例，分析整理的最佳實踐原則 (Best Practice)
  • 本文讓讀者了解如何針對 Cloud 設計出最適當的架構基本原則
  • 本文主要提供給技術角色閱讀，像是 CTOs、Architects、Developers、Operations

文件摘要如下：

  • Definition of the AWS Well-Architected Framework
  • General Design Principles
  • The Four Pillars (支柱) of the Well-Architected Framework
    • Security Pillar
    • Reliability Pillar
    • Performance Efficiency Pillar
    • Cost Optimization Pillar

Definition of the AWS Well-Architected Framework

General Design Principles

  • Stop guessing your capacity needs: 別再猜測到底需要多少機器 (容量) 才可以滿足業務需求，Cloud 天生就是要解決這樣的。
  • Test systems at production scale
  • Lower the risk of architecture change
  • Automate to make architectural experimentation easier
  • Allow for evolutionary architectures
