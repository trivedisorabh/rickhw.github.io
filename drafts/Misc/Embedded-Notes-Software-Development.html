Cross Platform Development

  • Cross platform development
  • Target board 資源有限：CPU, memory, storage 能力都有限, 所以不建議在 target 開發
  • 不容易在開發平台找錯誤

基本的動作:

  1. 在 host 寫出 .c, .h code
  2. 透過 cross compile, 將 code 編譯成可以讓 target 執行的 machine code, 然後包成影像檔 (image) 或可執行檔 (execution)

Cross Compiler

  • compiler 編譯 C code, 產生出
  • assembler:
  • linker: symbal table mapping
  • debugger
  • tools

上述總稱 embedded development toolchain (嵌入式工具箱), 一般簡稱 toolchain.

2. 把程式放到 target board

有三種方式

  1. 把 IC 放到燒錄機燒錄
  2. 透過 JTAG (IEEE 標準)
    • 可以直接透過協定，直接控制硬體
    • 把編譯好的 binary 傳到板子，透過硬體指令燒寫
    • 有成本問題, 有 JTAG 比較貴
  3. 仰賴軟體的方式燒錄
    • target board 必須要有現成的程式, 透過它來燒寫, 像是 bootloader.
    • 這個 program 可以驅動基本的周邊介面, 包含 usb, network, i/o, serial port 然後和外界取得溝通.

嵌入式板子通常有: CPU, Memory

  • CPU: 開機要只到一個 addressable 的 ROM
    • 由設計者來指定開機的位址
    • CPU 定址匯流排, 經過解碼電路設計, 可以指定到 ROM 存放程式的位址. 相關課程計算機組織
  • ROM, EPROM
  • Flash 分兩種
    • NOR: 可以定址, 用來存程式的, 讓 CPU 開機來指定
    • NAND: 當作大拇哥, 無法被定址
  • RAM: 關機就不見了

3. Debug Program

一般的 debug 的方法：

  1. printf
  2. 透過 debugger: break point, 看 memory, register

嵌入式的程式是跑在 target,

  1. 告訴 target, output 往哪裡丟, 像是 tty0. 將 printf 倒到.
  2. 透過 JTAG / serial 把資料網外界丟
  3. 在 target board 上跑一個 debugger (在跑 hello 之前), ex: gdb hello
    • debugger 怎麼顯示訊息？
    • 常見的方法就是將訊息透過 UART/serial port, 傳給 host machine.
    • 彷彿一般在 local 開發 application 一樣
  4. ICE (in-curile emulator):
    • ICE 和 CPU 有關, 通常和板子要搭配使用
    • 也有一些 ICE 是稱生 Multi-ICE
  5. IDE: Ingetrated Development Environment
    • Code Editor
    • Compiler
    • Debugger

嵌入式的開發環境的選擇通常 板子, ICE, IDE 通常要一起考慮.

Bootloader

在嵌入式系統叫做 bootloader，在 PC 就是 BIOS 或者 UEFI。

Bootloader 決定作業系統的開機方法，但是在嵌入式系統世界裡，每個板子的 bootloader 都是不一樣的。所以嵌入式系統有一部分的工作都是 bootloader porting。

bootloader 可以驅動一些基本的裝置，像是 Ethernet、USB，然後可以下載程式，然後更新作業系統、或者自己。

IPL (Initial Program Load)

可以啟動、載入、執行作業系統的裝置

MCU (微控制器)

微控制器 (MCU)，全名為 Microcontroller Unit，一顆完整的 MCU，會將 CPU、RAM、ROM、I/O 或 A/D 等周邊關於記憶與運算功能整合在一起，簡單來說，MCU 是一個微型電腦。
