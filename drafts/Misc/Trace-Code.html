Trace Code

最近常在 trace code, 把一些 trace 的心得記錄下來. 大概幾個步驟:

  1. 執行
  2. 改設定
  1. 找到進入點，確定可以在 IDE 跑起來。
  2. 找到 Settings / Config 檔，可能是 .properties、.xml、.plist、.inf、.conf … whatever .. 看懂他
  3. 看 Interface 和繼承架構, C/C++ 就看 header file
  4. 看看用了哪一些 third-party api/libraries, 如果是 secondary 的東西, 先拔掉.

執行

知道原本的程式怎麼跑起來, 不管是透過 ide or make, whatever …

抓架構

瘦身

  1. 拔掉 third-party libraries
  2. 根據一些 pattern，拔掉看不懂的，如果是 command pattern, 那橫向的東西 (通常會放在同一個 package) 就不用客氣了!
