Code Signing (簽署程式)

有簽署過的 ps1 才能夠被執行

簽署文件: user 憑證範本登記的憑證
簽署程式: 指令碼需要發行者 => Code Signing 憑證範本登記的:

  1. 發行 Code Signing 憑證範本到 CA
  2. 要簽署程式, 指令碼的人登記憑證
  3. 用合適的工具簽屬, 以 PowerShell 的 script 為例:
    $Cert=dir cert:\CurrentUser\my -CodeSigningCert
    Set-AuthentiCodeSignature $Cert
  4. 用戶執行程式, 指令碼前檢查簽章狀態
    Set-ExecutionRpolicy AllSigned -Force

有簽署過後的 ps1, 內容會增加簽署資訊.
如果程式被異動過, 就無法執行.

AD RMS

  • AD RMS 用戶帳號 (人或群組 必須 mail 屬性有值
  • 服務帳號, 不可與管理者帳號相同, 但都必須是 Domain 上的 user 帳號
  • 用戶存取授權 (CAL) 不含在 Windows Server 的 CAL
  • 用戶與 AD RMS 叢集通訊可使用 http or https
  • 若使用 http 則完全不需 CA, AD RMS 的 XRML, 不是 X.509 (CA 發的).
    XRML 由 AD RMS Root Cluster 發的.
