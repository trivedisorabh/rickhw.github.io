About DevOps Engineer

幾個月前在找 DevOps Engineer 這樣的人，找人就要先定義角色，這是一個工作內容難以定義的職務，問十個人，可能每個人的認知都不太一樣，特別是跟老闆溝通的時候。

有人覺得 DevOps 是一種文化，不是一種角色，這句話我只認同一部分，因為文化很重要，角色定義也很重要。

整理我認為什麼是 DevOps Engineer，以及為什麼我認爲 DevOps is a culture, also a role.

軟體工程生命週期 (Software Development Lifecycle, SDLC)

Google 關鍵字：Software Development Lifecycle, SDLC 可以找到找到很多圖，說明開發生命週期的各個階段，很容易找到很多階段的圖，為了在組織裡容易溝通，我通常會用四個階段來表達這件事情：Plan 、 Development 、 Testing 、 Operation

這四個 階段 有幾個面向的意涵：

  • 非時間序列：

更抽象的說法

  • Plan -> Excution -> Acceptance -> Operation

這個職務做什麼？

我定義的如下：

  • Resource Provisioning on Cloud
  • Deployment Methodologies
  • Observation
  • Network Infrastructure and Security Operations
  • CI / CD Flow and Pipeline
  • Collaboration System

這些事情有部分，在很多新創公司裡有一個特性：都是 CTO / Team Leader / 老闆自己做，這些系統或者是流程規劃的內容，通常也都是的三不管地帶，換言之，通常只做過一次，就是安裝的第一次。

怎麼說呢？

像是 Resource Provisioning 可能做完一次之後，就再也不會重新做了，因為累積很多東西，沒人敢動、不敢砍掉重練、不知道到底裝了什麼東西。

部署的方法，就是那一套，萬年唯一，殊不知，部署的方法要隨著需求改變而改變。像是一開始就導入 Rolling Deployment，然後就一直用這樣的方法在走。

監控系統 (Observation)：大部分都用固定的技術，像是 Cacti / Nagios / CloudWatch …

有時候負責的人會叫一個剛畢業的小朋友做，最後小朋友要走了，或者已經走了一段時間，才發現這些東西根本沒人知道設定是怎麼回事。

可能會是從這些角色過來

  • 從傳統的 IT / Infrastructure / System Engineer 轉過來的
  • 從 Software Engineer / Developer 轉過來的
  • SDET / SQA

必要的條件

  • 了解軟體開發流程，不管是傳統的 waterfall、agile、MVC、XP … 都可以。
  • 至少熟悉以下程式語言各一種：
    • 高階強型語言：C, Java, C#, Go …
    • 高階動態語言：python, javascript, perl, ruby …
    • script：bash, powershell, bat .
    • SQL, HTML
  • 熟悉網路基本概念以及相關應用服務
    • TCP / IP / UDP
    • HTTP / HTTPS / SSL
    • NAT / VPN / BGP
    • LDAP / Window AD/DC
    • DNS / SMTP / Load Balance
    • HTTP Server
  • 熟悉 Unix / Linux / Windows / OS X 等系統管理
    • init procedure
    • system monitor / resource
    • file system, network config

很多嗎？其實大部分都滿基本的。

Skill Chains

  • Provisioning
    • AWS CloudFormation
    • Terraform
    • Container / Docker / K8s / ECS / Rancher
  • CI/CD:
    • AWS
      • CodeDeploy
      • Beanstalk
      • OpsWorks
    • Chef
    • Ansible
  • Observation
    • AWS CloudWatch
    • Prometheus / InfluxDB
    • Fluentd, Collectd, Telegraf
    • 傳統
      • nagios
      • Cacti
