淺談 Microservice

最近在 Study API Gateway 相關文章，同時也在評估 Solutions。

略提 Microservice 的前提

Microservice 的透過架構 解耦 (decouple) 架構，重新定義 最小顆粒度 (granularity, fine-grained)，達到系統可以 快速反應 商務需求，同時又可以滿足可靠性 (Reliability)、擴展 (Scalability)、效能 (Performance)、容易部署、

以下整理我覺得導入微服務之前，要清楚之內將會面對的問題：

  1. 架構挑戰：微服務天生就是 分散式架構，所以將會面對高複雜度的協作、開發、系統性問題。整理以下：
    • 資料一至性：CAP 定理
    • 可靠性 (Reliability)
    • 擴展 (Scalability)
    • 效能 (Performance)
    • 安全 (Security)
    • 開發與測試流程
    • 部署
    • 成本 (Cost)
  2. 導入必要前提: 參考 Martin Fowler 提到的 Microservice Prerequisites
    • Rapid provisioning: 我整理的心得文 Resource Provisioning and DevOps
    • Basic Monitoring: 相關心得文 淺談系統監控與 CloudWatch 的應用, Monitoring vs Observability, What is Monitoring
    • Rapid application deployment: 快速部署應用程式，這邊提到的就是如何把 CI / CD 做好
    • 基礎架構自動化: 也就是 Infra as Code
  3. 維運管理
