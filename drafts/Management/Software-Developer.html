Senior Software Developer

我個人的職涯目標以及最想要的工作職位是：Software Architect，但是有一個 title 也是終身的，就是：Software Developer。一般都會分初階 (Junior) 和進階 (Senior)。那什麼樣的條件才是可算是一個資深軟體開發者？

本文以 軟體開發者 (Developer) 代替 軟體工程師 (Engineer)

參考了 Programmer Competency Matrix 的分析，同時也加入我個人的一些想法：

  • 能夠駕馭的程式語言
  • 程式語言以外的東西
  • 分析與設計
  • 測試
  • 工具
  • 電腦基礎科學
  • 軟體工程與方法
  • 自我學習與管理
  • 溝通與表達

能夠駕馭的程式語言

身為一個軟體開發者，有一個以上熟悉的程式語言，絕對是理所當然的，一些超過十幾年的開發者，熟悉兩三個都是很正常的。所謂熟悉是指能夠快速掌握到精神，然後找出最適當的方式，用來解決我們所要面對的問題。著重於用語言解決問題的能力，了解語言的特性與性能、優點缺點。

駕馭語言如母語

  • 程式語法，知道語言特性，優缺點
  • 標準函式庫
  • 編譯運作原理
  • 熟悉的 Framework
  • 測試方法與工具
  • 與其他語言或者工具整合

開創

  • 能夠設計函式庫或者 Framework
  • 效能調教
  • 改善或者設計新的語言, ex: DSL

經驗

  • 有效率的 Debug 方法與工具
  • 寫出有意義的 log 與 trace message
  • 知道語言的歷史與缺陷
  • 寫出可讀性的程式碼
  • 知道程式效率
  • 處理過 memleak 與 dead-lock 問題
  • 知道怎麼有效的測試
  • 知道如何定義測試涵蓋率

程式語言以外的東西

  • Regular Expressions (RE)
  • Shell Script
  • Database
  • 系統環境，像是 Linux
    • 開機生命週期
    • 系統調整
  • 資訊安全
  • 網路架構

分析與設計

  • 抽象化
  • 生命週期
  • 角色分析
  • User case / User Story / User Scenario
  • Modularize (模組化)
  • Configurable
  • Testable
  • Design Pattern

電腦基礎科學

成熟的 Software Developer 最基本的就是熟悉手上的語言，不管是 Java / C / C++ / Objective-C / Python / RoR / Javascript …

但是大家都知道，所謂的 program = data structure + algorithm ，語言則是用來表達這兩者的工具。所以首先要熟悉的就是：

  • Data structure: collection, array, list, set, hash, B-Tree … etc.
  • Algorithm: sort, search, graph, tree

這兩門課不管是考試，還是一般工作的面試都是很常出現的。

結語

那天跟同事聊天聊到關於『用人』的看法，我們聊了三國演義的一些例子：

  1. 如果你是曹操，你會用呂布嗎？
  2. 如果你是諸葛亮，你會斬馬謖嗎？你會用魏延嗎？
  3. 怎麼用人？需要人要去哪找？

後來我回家路上，又想了幾個問題：

  1. 如果你是徐元直，你會終身留在曹營？
  2. 如果張遼，

這些問題也大概是我 interview 會問的東西。

結論

金庸小說在神雕俠侶裡，黃蓉常常無法猜到楊過的心思。金庸的描述是這樣：閱人無數的黃蓉，不管是什麼樣的江洋大盜、江湖打手，還是一幫之主，談判過程她都能很快的猜到對方的心思，然後誘之以利、動之以情、說之以理，幫助郭靖。

從這些描述不難知道，謀事者，除了自身的能力之外，視人之能

用我一直很喜歡的圖做結尾，也期許自己是那個能夠為世界做出一點貢獻。
