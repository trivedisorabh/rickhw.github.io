Repair mongo when server unexpected shutdown

Problem Description

The mongo cannot start when server unexpected shutdown

  • CentOS 6.5 in VMWare Funsion
  • Recover Data after an Unexpected Shutdown

I got following message:

Mongo cannot repair with exception: exception in initAndListen std::exception: locale::facet::_S_create_c_locale name not valid, terminating.

Solution

It caused by mongo is unable to work with operating system’s current locale reference.

I still cannot start with mongod start, even remove the lock file.

Start successfully by manual
