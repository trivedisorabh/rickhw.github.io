使用 連結號 (Dash) 當名稱

接觸微軟 (M$) 相關的工具程式 (e.g, PowerShell) 發現習慣用 [Dash, -][31] 當名稱的連結符號。

我寫程式會避免這樣使用，因為在大部分的程式語言裡面，Dash 通常是 運算子 (Operator)，他會被 compiler 檢查出來，即使是 interpreter (e.g, PHP, python, perl) 在 runtime 也會出現錯誤。

名稱使用的範圍

以下的東西通常都會避免出現符號，一般都會使用 underline or underscope, _ 連結單字：

  • 檔名: 實體檔案、Folder 名稱, 有時候檔案名稱在 runtime 會變成變數名稱
  • 變數: 包含 Class Name, Global Variable, Local Variable
  • 參數: Argements in Command Line, Parameters in Methods, Functions, 他是另一種變數
  • key: dict, map, properties ..
    • json, yaml, properties, xml … etc
  • URL query string
  • Database: DB Name, Column Name

最近的趨勢是這些會使用 dash 連接名稱：

  • 檔名: 實體檔案、Folder 名稱
  • 參數: Argements in Command Line
  • key in yaml or json

實驗：Dash 當成變數
