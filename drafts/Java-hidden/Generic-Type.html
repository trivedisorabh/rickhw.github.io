Java Generic Type

Generic (泛型) 是在資料結構上，某一些類型可以是 泛用類型，或叫做 通用類型，也就是可以不需要特別指定，而這些資料可以應用在 class member field、method、interface、constructor 等。

泛型最直接的好處就是可以 developer 省去很多檢查和轉型 (Cast) 的工作，寫出來的 code 更加簡潔。另外帶來的好處，就是因為減少轉型，所以也減少了在 runtime 產生的 ClassCastException，這些錯誤在 Compile time 就可以檢查出來。缺點就是從此 Java Code 裡就出現 HTML … XDDD

Java 5.0 開始支援泛型，我剛開始只有在用 Collection (Set、List、Map) 常會用到而已，並沒有深入瞭解，最近在 trace code 東西發現使用的其實已經很頻繁，所以整理 study 筆記在這裡。

Declare

針對 Class 做修飾，格式 class ClassName<TypeName1, TypeName2, ...>，常見的慣例是 class ClassName<T, E>，這樣表示在 Class 裡面某些地方會使用到一些泛用類型，底下是一些常看到的慣例以及說明：

  • T 通常表示 Type, E 表示 Element, K 表示 Key, V 表示 Value
  • Generic type name 可以自行定義，規則同 Java naming rule
  • 名稱的範圍就是在 class 裡
  • 可以宣告多個，每個名稱用逗點隔開，有時候會看到這樣的宣告：class SocketChannel<A, B, C, D>
  • 除了一般的類別，泛型也可以在 interface 使用。
  • Java 是單一繼承，所以宣告 ClassName<T> 也等同於 ClassName<T extends Object>

這些名稱只是為了避免混淆，所以大部分都會用 T、E、A … 一個大寫字母代表，底下的例子也是可行的，我只是故意取得比較長而已。

Bound Type

Bound Type 是用來限制可使用的型別範圍，使用上就是：

  • Upper Bound：<T extends super-class>，表示繼承的子類別都是合法的
  • Lower Bound：<T super child-class>，和上述相反，這個父類別都是合法的

實際上在 Eclipse 裡用 Lower Bound 宣告 class 會 compile error.

Upper Bound: Extends

Wildcard

Wildcard 問號 ? 表示通用類別，wildcard 在電腦程式領域的意思，大概就是用一個符號代表一個集合，像是在 DOS 或者 unix command line 常會用 * 代表所有的；Regular Expression 用 . 代表任意一個字元、SQL 則是用 %。

結論

Java 和 C 一樣，屬於靜態語言 (Static Language)，使用任何變數之前，必須先講清楚這個變數是哪一種資料型態，是整數 (Integer)、字串 (String)、時間 (Date) … 等。如果沒有講清楚，那麼在編譯時期 (Compile Time) 就會出錯。但是有些時候需要 Meta 的描述，或者根本不知道 Type 的時候，這種有 動態 感的 Generic 就派得上用場了。

寫這段字的時候，我同時在猜想，像 scala, groovy 這種動態語言，差不多都是在 JDK 5.0 之後出現。而 Java 從一個本來是 Strong 且 Static 的語言，變成軟硬通吃，更加圓融且成熟。當然這也可能是我個人一廂情願的想法 XDDD
