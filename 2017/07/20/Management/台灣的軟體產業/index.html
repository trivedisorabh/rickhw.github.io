台灣軟體產業的經營

看完 這段 有感而發。。。

原始影片：

台灣那麽多好的軟體人才，卻連一家像樣的軟體公司都沒有。二十多年來說來說去不就是 Trend Micro …. 沒了。。。其他都是外商，來沾醬油的。

台灣的『軟體公司』永遠只在做專案、砍成本、能凹就凹、忽略基礎工程、永遠只有眼前、能動就好、外行領導內行、外來和尚會敲鐘 ….. 了不起就是趕流行，喊喊一些名詞，DevOps、Agile、雲端 … 卻連基本的專業素養都沒有，也沒有對市場趨勢的靈敏度，做事的方法完全沒改變 …. 還在用鑽木取火，殺豬公等方法。

Amazon 搞 EC 可以搞出全球最頂尖的 Public Cloud - AWS，讓 M$ 咬牙切齒地動用全公司的資源、組織異動 (火掉 3000 人)，讓 Google 把箭靶全部指向 AWS ….

人家已經上太空，我們的老闆還在擁抱過去在台灣的成功，自我感覺良好，滿滿的水桶，什麼都倒不進去。。。

中國企業把軟體開發當作投資，投入 Open Source 的開發、貢獻，加入 Apache 基金會，直接參與 Open Source 的開發，正面影響全世界的軟體工程。台灣的著名的 Open Source 貢獻者說得出來的 唐鳳、黃敬群 …. 有企業重用他們了？知道他們是誰？台灣的老闆聽過這些人？知道他們多有影響力？還是只知道這個月財報賺多少？

軟體是靈魂，未來世界各行各業都有軟體的介入。商業目標只是肉體、性。台灣的老闆往往只看到肉體、性，只看業績，只看眼前的金錢跟數字，嗜血的要員工付出新鮮的肝，做出來的東西往往是沒有靈魂的行屍走肉、沒有用的垃圾，人才在台灣企業底下變瓦礫，不再是人才。

所以年輕人出走？要怪誰？要怪那些自以為很厲害的老闆們。

Talent 常被當成或者變成 Resource，有本事就把 Recourse 變成 Talent.

有本事的老闆要創造好的環境，好的舞台、好的方向，讓工程師揮灑他的才華，為企業帶來貢獻，為人類社會帶來福祉；同時提攜後輩，做個伯樂、培養後進。

沒本事的老闆，把這些都推給工程師，說你全部都要自己來、你要自己想目標，那要你這老闆做什麼？你除了錢，你還有什麼？
