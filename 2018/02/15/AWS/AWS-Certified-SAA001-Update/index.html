AWS SAA-001 Update (Feb, 2018)

AWS Certified Solutions Architect - Associate (以下簡稱 SAA) 更版，來自 AWS Blog 的訊息: New AWS Certified Solutions Architect – Associate Exam: Now in General Availability.

正在準備考試的、或者有參考 我的筆記 的人，要注意此資訊。

考試基本資訊

考試要知道的資訊整理如下：

  • exam guide: 考試說明 (PDF)
  • sample exam questions: 範例考題 (PDF)
  • practice exam: 線上試考 (Web)

這跟過往都差不多，但是可以掌握整個考試方向。

版差差異

AWS SAA 更版考試編號為 SAA-01。考試的方向也更改了，主要以 Well-Architected 五個 Pillars 為考試主軸，項目以及比重如下：

  • Designing resilient architectures - 34%
  • Defining performant architectures - 24%
  • Specifying secure applications and architectures - 26%
  • Designing cost-optimized architectures - 10%
  • Defining operationally excellent architectures - 6%

比重以 resilient 的最高、Security 和 Performant 次之。

比重上看來，不難猜測接下來 SysOps 更版會偏重在 Cost-optimized, Operationally execllent.

Whitepaper 數量變少了，建議閱讀從原本九篇變成以下兩個部分：

  • Architecting for the Cloud - Best Practice - 以 Feb 2016 版為主
    • 同樣標題，還有另一個 Jan 2011 年版的
  • AWS Well-Architected: 這是 2016 re:Invent 加入第五個 Pillar 之後，新增的官方架構頁面，深度提供每個 Pillar 的資訊以及實務案例。

實際上的數量，光是 Well-Architected 頁面是多到不行，所以第一眼看到只剩下兩個項目，以為數量變少了，實際上是更多了 XD

剛好不久前，我在公司內部教育訓練，導讀過這兩篇 Whitepaper。其中一篇請參閱： Study Notes - Architecting for the cloud

正在準備的人

整體來說，比前一個版本，範圍更大 + 深度更深。

建議把準備方向：

  • 熟讀 Whitepaper: Architecting for the Cloud - Best Practice
  • 閱讀 AWS Well-Architected Framework 的資料:
    • 五個 Pillar 都要熟讀，每個都要考量 Design Priciples, Best Practice … 等
    • Whitepaper 裡都有描述對應的 AWS Services，以此做 Lab，整理自己的心得 (寫 Blog 是不錯的方法)
  • 多做 Lab: 這是一定的。
  • Youtube 的資源：
    • AWS - Deep Dive: 根據 5 Pillars 找到的 AWS Services，找到對應的 Deep Dive 看。
    • This is my architecture

正在準備的怎麼辦？

公告上有提到，到今年八月 (Aug, 2018) 都還可以考前一個版本。

我的準備心得 應該還是有參考價值 XD

考過了就是 Solution Architect (SA)?

想想這些問題：

  • 唸過資料結構、演算法就真的能寫出好的程式？
  • 唸過恐龍書就寫得出 OS？就會變成 Linux or Windows 專家？
  • 有碩士學位 (Master Degree) 就是 Master (大師) ？

我想的是這樣：

  • 準備考的過程，自己獲得的最多。
  • 考試可以讓自己知道不足的在哪裡，可以看到整個大的世界 (AWS 東西到底有多多)
  • 學習為什麼人家會這樣想，為什麼我們這樣做？
  • 反思：我們的系統缺什麼？怎麼走過去？
  • 下次再做的時候，要怎麼做？如果還有下次

至於有沒有考過，那是另一回事。台灣教育體制之下，培養出很多很會考試的人，這些人的專長就是考試，再難的都能考。。。但是不會獨立思考、判斷、組織，更別探討本質、想像力、推理之類的。

考試一向不是我擅長的，但我在考 AWS 三張考試過程中，獲得很多不一樣的反思，像是 本質性的思考、職涯的定位、學習的方法 … 這些其實是我覺得最有價值的。這些反思與獲得，都寫在準備心得裡：

  • 2016/08/20: AWS Certified Developer - Associate 準備心得
  • 2016/07/29: AWS Certified SysOps Administrator - Associate 準備心得
  • 2016/04/22: AWS Certified Solutions Architect - Associate 準備心得

最後，最近的心得依舊是底下這些，都是我的文章裡常出現的句子：

  • 努力不一定有結果，但不努力，一定不會成功。
  • 努力過了，再來講天賦。因為大部分人努力程度之低，根本輪不到拼天賦
  • 世上最可怕就是：比你聰明的人，比你還努力還認真。
  • 十年才剛入門而已
  • 才能這種東西，本來就是該靠自己挖掘創造的

給同樣正在這個行業努力、奮鬥的朋友。
